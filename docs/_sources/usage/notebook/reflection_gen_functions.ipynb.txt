      "def alg_lso_cluster(name, graph, **kwargs):\n",
      "    '''\n",
      "\n",
      "    A wrapper of *lso-cluster* algorithm from https://github.com/twanvl/graph-cluster \n",
      "    \n",
      "    Please specify long-format options. Don't specify output.\n",
      "\n",
      "    Arguments\n",
      "        Optional parameters regarding the clustering:\n",
      "        \n",
      "        ==========    ====================================================================================\n",
      "        'eval',         clustering Evaluate the objective function on clustering, do not optimize.\n",
      "        \n",
      "        'init',         clustering Use the given clustering as initial value for the optimization. Default: each node is initially in a separate cluster, i.e. clustering=1:length(A).\n",
      "        ==========    ====================================================================================\n",
      "                \n",
      "        Optional parameters regarding the objective:\n",
      "\n",
      "        ====================    ====================================================================================\n",
      "        'loss',                   loss Use the given loss/objective function. The loss is given a string name. See below for a list of supported loss functions. Default: loss = 'modularity'\n",
      "        \n",
      "        'total_volume',           m Replace the total volume (sum of edges) by m. Many objectives use the total volume for normalization, and changing it will change the scale at which clusters are found. Usually increasing the total volume will result in larger clusters. Default: m = sum(sum(A))\n",
      "        \n",
      "        'extra_loss',             alpha Add a term to the loss function that penalizes the volume of clusters, with weight alpha.\n",
      "        \n",
      "        'num_clusters',             n Force the solution to have the given number of clusters. The algorithm uses a binary search to alter the objective until it finds a solution with the given number of clusters. The alteration is the same as the one used by extra_loss.\n",
      "        \n",
      "        'min_num_cluster',         n Force the solution to have at least the given number of clusters.\n",
      "            \n",
      "        'max_num_cluster',         n Force the solution to have at most the given number of clusters.\n",
      "        \n",
      "        'max_cluster_size',         n Allow clusters to have at most n nodes.\n",
      "        ====================    ====================================================================================\n",
      "        \n",
      "        Optional parameters about internal algorithm details, you only need these if you know what you are doing:\n",
      "\n",
      "        ========================    ====================================================================================        \n",
      "        'seed',                         random_seed Use a given random seed. By default a fixed seed is used, so repeated runs with the same input give the same output.\n",
      "        \n",
      "        'num_repeats',                 n Repeat the search n times from scratch with different random seeds and return the best result. Default: 1\n",
      "        \n",
      "        'num_partitions',             n Number of times to try and break apart the clusters and re-cluster them Default: 0\n",
      "        \n",
      "        'optimize_exhaustive',         bool Use an exhaustive search instead of local search. This is of course very slow. Can be combined with max_num_cluster. Default: false.\n",
      "        \n",
      "        'optimize_higher_level',     bool Use a hierarchical optimizer, where small clusters are considered as nodes of a higher level graph. Default: true.\n",
      "        \n",
      "        'always_consider_empty',     bool Always consider the move of a node into a new singleton cluster. Default: true.\n",
      "        \n",
      "        'num_loss_tweaks',             n Maximum number of iterations in the binary search to force the specified number of clusters. Default: 32\n",
      "        \n",
      "        'check_invariants',             bool Check invariants of the algorithm (for debugging). Default: false.\n",
      "        \n",
      "        'trace_file',                 filename Write a trace of the steps performed by the optimization algorithm to a file in JSON format.\n",
      "        \n",
      "        'verbose',                     level Level of debug output. Levels go up to 7 and are increasingly chatty. Default: level = 0, i.e. no output.\n",
      "        ========================    ====================================================================================        \n",
      "    \n",
      "        Some of the supported loss functions are:\n",
      "\n",
      "        ================    ====================================================================================                    \n",
      "        'modularity',         loss = -sum_c (w_c/m - v_c^2/m^2) This is the negation of the usual definition\n",
      "        \n",
      "        'infomap':             The infomap objective by [3].\n",
      "        \n",
      "        'ncut':                 Normalized cut, loss = sum_c (v_c - w_c) / n_c\n",
      "        \n",
      "        'rcut':                 Ratio cut, loss = sum_c (v_c - w_c) / v_c\n",
      "        \n",
      "        {'pmod',p}:             Modularity with a different power, loss = -sum_c (w_c/m - (v_c/m)^p / (p-1))\n",
      "        \n",
      "        {'mom',m}:             Monotonic variant of modularity, loss = -sum_c (w_c/(m + 2v_c) - (v_c/(m + 2v_c))^2)\n",
      "        \n",
      "        'w-log-v',             loss = sum_c (w_c/m * log(v_c) )\n",
      "    \n",
      "        num loss = |C|:         Minimize the number of clusters, this leads to finding the connected components.\n",
      "        ================    ====================================================================================\n",
      "                    \n",
      "    Reference\n",
      "        Graph clustering: does the optimization procedure matter more than the objective function?; Twan van Laarhoven and Elena Marchiori; Physical Review E 87, 012812 (2013)\n",
